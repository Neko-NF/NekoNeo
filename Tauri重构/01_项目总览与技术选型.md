## 一、重构背景与目标

### 1.1 现有问题（Electron v1.x）

| 问题 | 具体表现 |
|------|---------|
| **包体积过大** | Electron 本体约 120MB，打包产物约 150MB+ |
| **内存占用高** | 常驻内存 150~300MB（含 Chromium 进程） |
| **主进程臃肿** | `main.js` 单文件近 2000 行，职责边界模糊 |
| **安全模型弱** | `nodeIntegration: true`，渲染层可直接访问 Node API |
| **PowerShell 依赖重** | 每次采集都 fork 子进程，延迟高且不稳定 |
| **可测试性差** | 主进程逻辑与 IPC 耦合紧密，无法单元测试 |

### 1.2 重构目标

- **轻量**：可执行文件 ≤ 10MB，常驻内存 ≤ 50MB
- **现代 UI**：简洁大方，拒绝堆砌效果，保持跨版本视觉一致性
- **可维护**：前后端职责分离，模块边界清晰，可测试
- **安全**：前端仅通过定义好的 Command 接口与后端交互

## 二、技术栈选型

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                   Tauri 2 应用进程                       │
│                                                         │
│  ┌─────────────────┐        ┌──────────────────────┐   │
│  │   前端（WebView）  │◄──────►│   Rust Core（后端）   │   │
│  │                 │  invoke │                      │   │
│  │  Vue 3 + Vite   │  events │  Commands / Services │   │
│  │  TypeScript     │        │  系统 API / HTTP 通信  │   │
│  └─────────────────┘        └──────────────────────┘   │
│                                     │                   │
│                              ┌──────▼──────┐            │
│                              │   OS / WinAPI │           │
│                              │  (无 PS 子进程) │          │
│                              └─────────────┘            │
└─────────────────────────────────────────────────────────┘
```

### 2.2 技术选型明细

| 层级 | 技术 | 版本要求 | 选型理由 |
|------|------|---------|--------|
| **应用框架** | Tauri 2 | `^2.x` | 轻量（Rust 后端）、安全（无 nodeIntegration）、系统 API 丰富 |
| **前端框架** | Vue 3 | `^3.4` | Composition API + `<script setup>` 语法简洁，学习曲线低 |
| **构建工具** | Vite | `^5.x` | 极速 HMR，天然支持 Vue 3 |
| **语言** | TypeScript | `^5.x` | 类型安全，IDE 提示，减少运行时错误 |
| **状态管理** | Pinia | `^2.x` | 官方推荐，支持 Devtools，模块化 Store |
| **路由** | Vue Router | `^4.x` | Hash 路由，与 Tauri WebView 兼容 |
| **样式** | 原生 CSS + CSS Variables | — | 无额外依赖，设计 Token 驱动 |
| **图标库** | Phosphor Icons (Vue) | `^2.x` | 与现有 Electron 版本一致 |
| **HTTP（前端）** | `@tauri-apps/plugin-http` | — | 走 Rust 层发请求，绕过 WebView CORS |
| **Rust 依赖** | 见下表 | | |

### 2.3 Rust Crate 选型

| Crate | 用途 |
|-------|------|
| `tauri` `^2` | 核心框架 |
| `tauri-plugin-autostart` | 开机自启 |
| `tauri-plugin-single-instance` | 单例运行 |
| `tauri-plugin-updater` | 自动更新（stable/beta） |
| `tauri-plugin-store` | 配置持久化（JSON） |
| `tauri-plugin-notification` | 系统通知 |
| `tauri-plugin-http` | 统一 HTTP 客户端 |
| `windows` `^0.58` | WinAPI 直接调用（窗口/电池/空闲/图标） |
| `sha2` | 设备指纹 SHA256 |
| `serde` / `serde_json` | 序列化/反序列化 |
| `tokio` | 异步运行时 |
| `image` | 截图模糊处理（像素化） |
| `screenshots` | 屏幕截图（跨平台，Windows 优先） |
| `reqwest` | HTTP 客户端（Rust 层状态上报） |

## 四、核心功能范围（v2.0）

### ✅ 保留功能

| 类别 | 功能 |
|------|------|
| **核心上报** | 定时状态上报、活动窗口检测、电池信息、空闲检测、媒体信息(SMTC)、应用图标、设备指纹、看门狗 |
| **截图上报** | 屏幕截图、独立截图间隔、截图大小限制 |
| **隐私保护** | 隐身模式、作用域控制、全局/按进程模糊、可视化进程选择器 |
| **开机自启** | 开机自启开关、自启最小化、自启延迟、自启时开启上报 |
| **系统托盘** | 托盘图标、右键菜单、关闭行为配置 |
| **HTTP 通信** | 状态上报 V2、配对握手、连通性测试、密钥验证、用户认证 |
| **自动更新** | stable + beta 双通道、强制更新弹窗、跳过版本、自动下载 |
| **UI** | 实时系统指标、主题色切换（亮/暗/跟随系统）、界面缩放、字体选择、日志控制台、设置页、关于页 |
| **系统** | 单例运行锁、系统通知 |

### ❌ 删除功能

| 功能 | 删除原因 |
|------|---------|
| SRS 直播推流（全部） | 与轻量化目标冲突，独立为后续模块 |
| OBS WebSocket 控制 | 同上 |
| Nightly 更新通道 | 简化，仅 stable + beta |
| 玻璃拟态 UI 效果 | 重构 UI，走简洁现代风格 |
| Electron 缓存清理 | Tauri 无此冗余缓存 |
| 本地测试账户 | 统一走服务端认证 |
| 双重认证 2FA | 未实现的占位配置，暂不做 |
| 仪表盘拖拽网格 | 重构为静态简洁布局 |
