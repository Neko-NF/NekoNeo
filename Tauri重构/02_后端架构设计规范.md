## 一、总体原则

1. **Commands 是唯一入口**：前端只能通过 `invoke()` 调用已注册的 Tauri Command，不能直接访问任何系统资源。
2. **Services 不依赖 Tauri**：`services/` 下的业务逻辑是纯 Rust 代码，不 `use tauri::*`，可独立单元测试。
3. **错误统一化**：所有 Command 返回 `Result<T, AppError>`，前端收到的永远是结构化错误对象，不是 panic。
4. **状态隔离**：全局共享状态通过 `tauri::State<T>` 注入，禁止使用裸 `static` 变量（除非 Mutex 保护）。
5. **异步优先**：涉及 IO（网络、文件、系统 API）的操作全部 `async`，避免阻塞主线程。

---

## 二、分层架构

```
前端 (Vue)
    │  invoke("cmd_name", { ...args })
    ▼
┌─────────────────────────────────────┐
│  Commands 层  src-tauri/src/commands/ │  ← 参数校验、权限检查、调用 Service
└─────────────────────────────────────┘
    │  调用
    ▼
┌─────────────────────────────────────┐
│  Services 层 src-tauri/src/services/ │  ← 核心业务逻辑，纯 Rust，可测试
└─────────────────────────────────────┘
    │  调用
    ▼
┌─────────────────────────────────────┐
│  系统层  WinAPI / reqwest / 文件 IO   │  ← OS、网络、磁盘
└─────────────────────────────────────┘
```

## 四、Services 规范

### 4.1 StatusReporter（核心上报服务）

```rust
// src-tauri/src/services/status_reporter.rs

use tokio::sync::Mutex;
use std::sync::Arc;

#[derive(Clone)]
pub struct StatusReporter {
    inner: Arc<Mutex<ReporterInner>>,
}

struct ReporterInner {
    running: bool,
    interval_sec: u64,
    consecutive_failures: u32,
    auto_restart_count: u32,
    last_result: Option<TickResult>,
    // 取消句柄
    cancel_tx: Option<tokio::sync::oneshot::Sender<()>>,
}

impl StatusReporter {
    pub fn new() -> Self { ... }

    pub async fn start(&self, config: &AppConfig) -> Result<(), AppError> {
        let mut inner = self.inner.lock().await;
        if inner.running { return Ok(()); }
        let (tx, rx) = tokio::sync::oneshot::channel();
        inner.cancel_tx = Some(tx);
        inner.running = true;
        let self_clone = self.clone();
        tokio::spawn(async move {
            self_clone.run_loop(rx).await;
        });
        Ok(())
    }

    pub async fn stop(&self) { ... }

    async fn run_loop(&self, cancel: tokio::sync::oneshot::Receiver<()>) {
        // 核心循环：每 interval_sec 执行一次 tick()
        // 看门狗逻辑内嵌：连续失败 >= 3 次触发 auto_restart
    }

    async fn tick(&self) -> Result<TickResult, AppError> {
        // 1. 读取配置（deviceKey, enableScreenshot 等）
        // 2. 调用 SystemInfo::get_active_window()
        // 3. 调用 SystemInfo::get_battery()
        // 4. 调用 SystemInfo::get_idle_ms()
        // 5. 调用 SystemInfo::get_media_info()
        // 6. 调用 Screenshot::capture() 并应用隐私模糊
        // 7. 调用 ApiClient::report_status_v2()
        // 8. 处理响应（KEY_REVOKED / DEVICE_NOT_FOUND / 429 等）
        // 9. 更新 last_result
    }
}
```

## 五、错误处理规范

### 5.1 统一错误类型

```rust
// src-tauri/src/errors.rs

use serde::Serialize;
use thiserror::Error;

#[derive(Debug, Error, Serialize)]
#[serde(tag = "code", content = "message")]
pub enum AppError {
    // 配置错误
    #[error("配置读取失败: {0}")]
    ConfigError(String),

    // 网络错误
    #[error("网络请求失败: {0}")]
    NetworkError(String),

    #[error("设备密钥无效")]
    InvalidKey,

    #[error("密钥已被撤销")]
    KeyRevoked,

    #[error("设备不存在")]
    DeviceNotFound,

    #[error("请求频率过高，请 {retry_after}s 后重试")]
    RateLimited { retry_after: u64 },

    // 系统错误
    #[error("系统 API 调用失败: {0}")]
    SystemError(String),

    #[error("截图失败: {0}")]
    ScreenshotError(String),

    // 更新错误
    #[error("更新检查失败: {0}")]
    UpdateError(String),
}

// 让 Tauri 能将 AppError 序列化传递给前端
impl From<AppError> for tauri::ipc::InvokeError {
    fn from(e: AppError) -> Self {
        tauri::ipc::InvokeError::from_anyhow(e.into())
    }
}
```

### 5.2 前端错误处理约定

前端 `invoke()` 捕获错误时，结构为：
```typescript
interface AppError {
  code: string;   // e.g. "KeyRevoked" | "NetworkError" | "DeviceNotFound"
  message: string;
}
```

## 七、事件推送（后端 → 前端）

后端通过 `app_handle.emit()` 主动推送事件，前端用 `listen()` 订阅：

| 事件名 | 触发时机 | payload 结构 |
|--------|---------|-------------|
| `service:tick` | 每次上报完成 | `{ success, appName, batteryLevel, hasScreenshot, timestamp }` |
| `service:log` | 上报服务产生日志 | `{ level, message, time }` |
| `service:status` | 服务启停时 | `{ running: boolean }` |
| `service:key_status` | 密钥状态变化 | `{ code, message }` |
| `metrics:update` | 系统指标采集完成 | `SystemMetrics` |
| `update:available` | 发现新版本 | `{ version, mandatory, releaseNotes }` |
| `update:progress` | 下载进度 | `{ percent, downloadedBytes, totalBytes }` |

```rust
// 后端推送示例
app_handle.emit("service:tick", &tick_result).ok();

// 前端订阅示例（Vue Composable）
import { listen } from '@tauri-apps/api/event';

const unlisten = await listen<TickResult>('service:tick', (event) => {
  serviceStore.updateLastResult(event.payload);
});
```

```rust
use tauri_plugin_autostart::ManagerExt;

pub fn autostart_enable(app: AppHandle) -> Result<(), AppError> {
    app.autolaunch().enable().map_err(|e| AppError::SystemError(e.to_string()))?;
    config_set(app.clone(), "enable_auto_start", true.into())?;
    Ok(())

    app.autolaunch().disable().map_err(|e| AppError::SystemError(e.to_string()))?;
    config_set(app.clone(), "enable_auto_start", false.into())?;
    Ok(())

    app.autolaunch().is_enabled()
```

## 十、隐私截图选择器（透明叠加窗口）

Tauri 2 支持创建透明、无边框、置顶的叠加窗口：

```rust
// src-tauri/src/commands/privacy.rs

#[tauri::command]
pub async fn privacy_open_picker(app: AppHandle) -> Result<Option<WindowInfo>, AppError> {
    // 1. 枚举可见窗口列表
    let windows = SystemInfo::list_visible_windows();

    // 2. 创建全屏透明叠加窗口
    let picker = tauri::WebviewWindowBuilder::new(&app, "privacy-picker", tauri::WebviewUrl::App("privacy-picker.html".into()))
        .transparent(true)
        .decorations(false)
        .always_on_top(true)
        .skip_taskbar(true)
        .fullscreen(true)
        .build()?;

    // 3. 传递窗口列表给前端（通过 emit）
    picker.emit("picker:windows", &windows)?;

    // 4. 等待前端结果（通过 oneshot channel + ipcMain 等效）
    // ...返回用户选中的 WindowInfo 或 None（Esc 取消）
    Ok(None)
}
```

## 十一、自动更新（tauri-plugin-updater）

```rust
// src-tauri/src/commands/update.rs

use tauri_plugin_updater::UpdaterExt;

#[tauri::command]
pub async fn update_check(app: AppHandle, channel: String) -> Result<Option<UpdateInfo>, AppError> {
    // 根据 channel 选择不同的 endpoints
    let endpoint = match channel.as_str() {
        "beta" => "https://github.com/Neko-NF/Neko-Status-Desktop/releases/latest/download/latest-beta.json",
        _      => "https://github.com/Neko-NF/Neko-Status-Desktop/releases/latest/download/latest.json",
    };

    let updater = app.updater_builder()
        .endpoints([endpoint.parse().unwrap()])?
        .build()?;

    match updater.check().await? {
        Some(update) => Ok(Some(UpdateInfo {
            version: update.version.clone(),
            release_notes: update.body.clone().unwrap_or_default(),
            mandatory: update.body.as_deref().map(|b| b.contains("[MANDATORY]")).unwrap_or(false),
        })),
        None => Ok(None),
    }
}
```
